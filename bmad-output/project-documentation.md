## 3. Entry Points and Key Flows

### Application Bootstrap

| Entry File | Role |
|------------|------|
| `FPTUniRAG/Program.cs` | Single composition root — DB context, ~20 repository/service DI registrations, DataProtection, SMTP/RagIngestion/Stripe options binding, cookie auth, 3 authorization policies, Razor Pages page-route conventions, HTTP pipeline, endpoint/hub mapping. |
| `FPTUniRAG.BusinessLayer/GlobalUsings.cs` | Shared `using` directives for the business layer. |

### Primary Routing Layer

Three route surfaces, all mapped in `Program.cs`:
- **Razor Pages** (`app.MapRazorPages()`) — ~25 pages under `Pages/`, role-gated via `AuthorizePage`/`AuthorizeFolder` conventions (not `[Authorize]` attributes).
- **Minimal API endpoint groups** under `FPTUniRAG/Endpoints/`: `AdminAccountApiEndpoints`, `TeacherSubjectApiEndpoints`, `StudentChatApiEndpoints`.
- **SignalR hubs**: `/hubs/teacher-header-subjects` (`TeacherHeaderSubjectHub`), `/hubs/student-chat` (`StudentChatHub`), `/hubs/subscription-plans` (`SubscriptionPlanHub`).

### Authentication / Middleware Chain

`UseHttpsRedirection → UseRouting → [custom: no-cache headers for protected/auth pages] → UseAuthentication → [custom: force MustChangePassword redirect to /ChangePassword, or 403 for /api & /hubs] → UseAuthorization → MapStaticAssets/MapRazorPages/Map*Endpoints/MapHub`.

Auth scheme: cookie auth (`.FPTUniRAG.Authentication`, 7-day sliding expiration) with custom `AccountCookieAuthenticationEvents`. Three role policies: `AdminOnly`, `TeacherOrAdmin`, `StudentOrAdmin`.

### Background Workers / Queues

- `DocumentProcessingBackgroundService` (hosted service) — drains a singleton in-memory `IDocumentProcessingQueue` (`System.Threading.Channels`-based) **sequentially, one document at a time**, for the whole application. Recovers pending jobs on startup.
- `AdminAccountInitializationHostedService` — one-time admin account seeding on startup.

## 5. Conventions and Patterns

### Naming Conventions

| Artifact | Convention | Example |
|----------|-----------|---------|
| Files | PascalCase, one type per file, `.cs`/`.cshtml` | `StudentChatService.cs`, `TeacherUpload.cshtml` |
| Classes / Types | PascalCase; interfaces `I`-prefixed | `IStudentChatService` / `StudentChatService` |
| Functions | PascalCase, `Async` suffix for async methods | `RetrieveRelevantChunksAsync` |
| Constants | PascalCase (no `SCREAMING_CASE` observed) | `AccountClaimTypes.MustChangePassword` |

### Module Resolution

Standard .NET namespace-per-folder + `ImplicitUsings`; `FPTUniRAG.BusinessLayer/GlobalUsings.cs` centralizes a handful of shared `using`s for that project. No path aliasing (not applicable in C#).

### Concurrency Model

`async`/`await` throughout (services, repositories, `HttpClient` calls). Background work uses `IHostedService` + a single-consumer `System.Threading.Channels` queue for document processing — explicitly **not parallelized** (see Planning Notes).

### Error Handling Style

`Common/OperationResult.cs` — a shared result type returned by business-layer service methods (success/failure + message) rather than throwing for expected failure paths. Exceptions are still used for unexpected/infra failures (e.g., Tesseract process failures throw `InvalidOperationException`).

### Logging Approach

Built-in `Microsoft.Extensions.Logging` — `Console` and `Debug` providers registered explicitly in `Program.cs` (`builder.Logging.ClearProviders()` then `AddConsole()`/`AddDebug()`). No structured-logging library (Serilog/NLog) or external log sink detected.

### Configuration and Secrets

`appsettings.json` / `appsettings.Development.json` (env-selected via `ASPNETCORE_ENVIRONMENT`), bound to strongly-typed options via `IOptions<T>`: `SmtpOptions`, `RagIngestionOptions` (OpenRouter key/models/base URL, Tesseract toggle), `StripeOptions`. Docker Compose sources secrets from a local `.env` (see `.env.example`; a real `.env` and the `.keys/` Data Protection key directory are explicitly excluded from commits per `CLAUDE.md`).

### Test File Location Convention

`FPTUniRAG.Tests/<Domain>/<ServiceName>Tests.cs`, mirroring the business-layer folder structure (e.g. `FPTUniRAG.Tests/Rag/Chat/StudentChatProgressTests.cs`).

### Database(s)

|----------|------|-------------|--------------------|
| PostgreSQL 17 | Relational (with vector-embedding storage for RAG chunks) | Entity Framework Core 10 + `Npgsql.EntityFrameworkCore.PostgreSQL` | `AddDbContext<AppDbContext>` with `UseNpgsql(DefaultConnection)`; schema authored directly in `create_database.sql`, not EF Core migrations |

### Message Queues / Event Buses

In-process only: a singleton `IDocumentProcessingQueue` backed by `System.Threading.Channels`, consumed by `DocumentProcessingBackgroundService`. No external broker (Kafka/RabbitMQ/SQS) detected.

### Object / File Storage

Local filesystem only — `App_Data/teacher-uploads/<subject>/` (configurable via `StorageRoot`), mounted as the `uploads-data` volume in Docker Compose. No S3/Blob/GCS SDK detected.

### Third-Party Services

| SMTP (via MailKit) | Email | `MailKit` |
